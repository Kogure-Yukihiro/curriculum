■ ネットワーク層
IP（Internet Protocol）
  相手先のアドレス情報をもとにしてパケットを送信する機能
   インターネット層のプロトコルとなる

IPヘッダ
　IPパケットの冒頭部にある、IPバージョン・送信元アドレス・宛先アドレスなどの情報が格納された部分のこと。

ICMP（Internet Control Message Protocol）
　IP通信の疎通状態を調査したり、正常に通信できない場合に
  送信元ホストへエラー情報を返したりする、IP通信を補佐するためのプロトコル

ICMPメッセージ
　通信状態を確認するために使用されるプロトコルで、ICMPメッセージによってエラーの通知や通信状態の確認する。
　エラー通知に関するErrorメッセージと、問い合わせに関するQwertyメッセージの2種類がある。

・IPアドレス
　インターネット上に接続されたPCやスマートフォンなどの機器が持つナンバーのこと
　IPv4とIPv6の二種類がある。
_____

オクテット
　オクテットとはIPv4において8ビットずつ、カンマで区切られた部分を指します。

ネットワーク部
 ネットワークを識別するアドレス
 あるネットワークに割り当てられたIPアドレス郡の中で、そのネットワーク自身を指し示すアドレスとなります。

ホスト部
　コンピュータや各機器を識別するアドレス
　あるネットワークに割り当てられたIPアドレス郡の中で、ネットワーク内の個々のホストに割り当てられるアドレスとなります。

クラス
　IPv4には「クラス」というものが存在し、それぞれ「クラスA」「クラスB」「クラスC」に分けられます。
　クラスの上位ビット、アドレス範囲、ビット数、最大ホスト数を下図で説明していきます。

プライベートIPアドレス
　インターネットプロバイダに割り振られているIPアドレスが、インターネットに接続するパソコンなどの機器に割り振られるIPアドレスのことです。
　インターネット上で通信を行う場合は、必ず必要なIPアドレスになります。

グローバルIPアドレス
　インターネットのようなグローバルなものではなく、社内LANなど小規模なネットワークで使用されるIPアドレスです。
　プライベート上のものになるので、インターネット上では使うことはできず、プライベートネットワークの中でのみ使われます。
_____

ネットワークアドレス
　ホスト部の2進数のビットがすべて０のアドレスのこと
　ネットワークアドレスは、そのネットワーク自体を指すアドレスとして使用されます。

ブロードキャストアドレス
　ホスト部の2進数のビットがすべて１のアドレスのこと
　ブロードキャストアドレスは、あるセグメントに接続されている全てのホストにパケットを送信するためのアドレスとして使用されます。

ループバックアドレス
　クラスAアドレスのうち、「127」で始まるアドレスのことをループバックアドレスと呼びます。

サブネット
　IPネットワークを論理的に細分化したもののことである

サブネットマスク
　IPアドレスと同様、32ビットで構成され、IPアドレスとセットで使う
　IPアドレスのネットワーク部を「1」、ホスト部を「0」で表し、どこまでがネットワーク部かを示す値となっている

プレフィックス表記
　何ビット目までがネットワーク部かを簡易的にした表記のこと
　IPアドレスの記載に続いて、「/（ネットワーク部のビット数）」で表記される。

デフォルトゲートウェイ
　同一ネットワーク内のノードから外部ネットワークに属するノードと通信する際に出入り口となるデバイスやポートのこと
　実務ベースでは、適切な転送先が見つからない場合に使用するIPアドレスを指すことが多い

IPv6
　各コンピュータに割り振られるIPアドレスを128ビットのデータとして表現するもの
　最大約340澗(かん)台のコンピュータがインターネットに直接接続できます。
　これは事実上無限と言える数です。
　128ビットとなるため2進数で128での表記となりますが、16ずつで区切り16進数で表すのが基本とされています。
　また、IPv4はカンマで区切られていましたが、IPv6ではコロン区切りとなります。

サブネットプレフィックス
　IPv4におけるネットワーク部に相当します。
　基本的に前半64ビットがサブネットプレフィックスにあたります。

インターフェイスID
  IPv4におけるホスト部に相当します。
　通常のIPv6ノードは、インタフェイスIDをMACアドレスから算出します。
______

ルータ
　コンピュータネットワークにおいて、データを2つ以上の異なるネットワーク間に中継する通信機器
　ネットワーク層において、どのルートを通してデータを転送すべきかを判断するルート選択機能を持つ

コンソール接続
　ルータのコンソールポートとPCをコンソールケーブルで接続する方法

VTY接続
　物理的に離れたところからネットワークを通じて仮想的に接続する方法

AUX接続
　ルータのAUXポート(補助ポート)とモデムを接続し、電話回線経由で接続する方法

ルーティングテーブル
　ルーターやネットワーク接続されたコンピュータが持つ、個々のネットワークの宛先への経路の一覧を保持しているテーブル状のデータ構造である。

ネクストホップ
　次に転送する最も近いルータやインターフェイスのこと

ARPテーブル
ARPによって割り出したIPアドレスとMACアドレスの対応表。

ARP（Address Resolution Protocol）
IPv4アドレスから対応するMACアドレスを調べる機能

コアルータ
 基幹ネットワークにおいてメインで用いられるルータ
 通信キャリアやISPネットワークの通信を中継するために利用されるルータで、大量のデータを転送する必要があるため、高速なデータ転送が可能

センタールータ
 企業で中心的に利用されるルータ
 拠点となる本社や、データセンター等の企業内で中心となる場所を接続するためのネットワークを構成する

エッジルータ
 企業で利用されるルータ
 中心となる拠点を除く、営業所や支社等の拠点間を接続するネットワークを構成する
 
ブロードバンドルータ
 家庭環境やSOHO環境等、インターネット回線に接続することを目的として利用されるルータ

ルーティング
　異なるネットワークにパケットを送信する際に最適な経路を決めること

直接接続
 その名の通り、ルータが直接接続しているネットワークのルートのこと

スタティックルーティング
 管理者が宛先ネットワークへの最適なルートを手動で設定したルートのこと

ダイナミックルーティング
 ルーティングプロトコルにより動的に追加されるルートのこと

・ディスタンスベクタ型
ホップ数（=宛先に到達するまでに経由するデバイス数）をメトリックとし、経路を選ぶ方式
自身のルーティングテーブルに、隣接ルータから転送された情報をそのまま追加していく方法でルート情報のやりとりを行う
シンプルな探索方法であり、ルータ自身で記憶しなければならない情報が少ないため、ルータへの負荷は小さくなる
一方で、通れない経路が発生した際、ルートの再構成に時間がかかる、というデメリットも存在する
小規模ネットワーク向き

・リンクステート型
各ルータが全体の接続情報を見て、最適経路を選択する方式
ルータ同士でLSA（Link State Advertisement）という自身のネットワーク情報を交換し、それを用いてネットワーク全体を把握している
各ルータの負荷がかかるが、ルートの再構成にかかる時間は短く済む
大規模ネットワーク向き

・ハイブリッド型
ディスタンスベクタ型をベースに、リンクステート型を組み合わせた方式
Ciscoが独自に拡張した方式であるため、1つでも非Cisco社製デバイスが混じっている環境では使用できない

■ トランスポート層

コネクション型通信
　通信を確立するもの同士で連絡を取り合って制御を行う通信方式

ACK【ACKnowledgement】
 二者間の通信で、相手方に何らかの肯定的な応答を返す際に送られる信号やデータ、パケットなどのこと。
「接続要求を受理した」「データが正しく受信できた」といった内容を伝達するために用いられる。

コネクションレス型通信
　連絡を取り合って制御はせずに、相手が受け取る準備ができているかにかかわらず、送信してしまう通信方式

TCP (Transmission Control Protocol) 
　トランスポート層で動作するプロトコルで、コネクション型通信を行う。

スリーウェイハンドシェイク
　IPネットワーク上でTCPによる接続（コネクション）を確立するための手順。パケットの送受信を3回行う。

Telnet
　ネットワークに接続された機器を遠隔操作するために使用するアプリケーション層プロトコル。

HTTP(Hypertext Transfer Protocol)
　WebサーバとWebブラウザの間で、Web情報をやりとりするためのプロトコル

FTP（File Transfer Protocol）
　サーバとクライアントの間でファイル転送を行う際に必要となる通信プロトコル

SMTP【Simple Mail Transfer Protocol】
　インターネットなどのTCP/IPネットワークで標準的に用いられる、電子メール（eメール）を伝送するためのプロトコル

TCPヘッダ
　TCPパケットの先頭部分にある、制御情報などを一定の形式で記述した領域

UDP (User Datagram Protocol)
　トランスポート層で動作するプロトコルでコネクションレス型通信を行う。

RTP【Real-time Transport Protocol】
　TCP/IPネットワーク上で音声や動画のように連続するデータの流れをリアルタイムに伝送するための通信プロトコル

SNMP (Simple Network Management Protocol) 
　IPネットワーク上のルータやスイッチ、サーバ、端末など様々な機器をネットワーク経由で遠隔から監視・制御するためのプロトコル

DHCP【Dynamic Host Configuration Protocol】
　IPアドレス等のネットワークに接続するために必要な情報を自動的に割り当てるプロトコル
　クライアントサーバ型のプロトコルである

TFTP（Trivial File Transfer Protocol)
　特定のコンピュータ間でファイル転送する時に使用するアプリケーション層プロトコル

RIP（Routing Information Protocol）
　ディスタンスベクタ型のルーティングプロトコル。
　距離と方向を基準に最適ルートを決定する。
　メトリック(ルーティングにおいて、最適経路を選択する際に指標となるもの)はホップ数（経由するルータの数）の値が小さい経路が最適ルートとして決定される。

UDPヘッダ
　UDPパケットの先頭部分にある、制御情報などを一定の形式で記述した領域
